<Note>
  **HTTP Method:** `DELETE`
  **Endpoint:** `/api/v1/memories/delete/{memory_id}`
</Note>

## Overview

Delete a specific memory from your agent's memory bank by its unique identifier. Only memories owned by the authenticated user can be deleted. This operation is permanent and cannot be undone.

## Path Parameters

<ParamField path="memory_id" type="string" required>
  The unique identifier of the memory to delete
</ParamField>

## Response

Returns confirmation of the deletion operation.

## Examples

### Delete a Memory

<CodeGroup>

```python Python
import requests

memory_id = "mem_abc123"
url = f"https://api.agent-mind.com/api/v1/memories/delete/{memory_id}"
headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}

response = requests.delete(url, headers=headers)
result = response.json()
```

```bash cURL
curl -X DELETE https://api.agent-mind.com/api/v1/memories/delete/mem_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const memoryId = "mem_abc123";
const response = await fetch(`https://api.agent-mind.com/api/v1/memories/delete/${memoryId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const result = await response.json();
```

</CodeGroup>

## Important Notes

- Deletion is **permanent** and cannot be undone
- Only the owner of a memory can delete it
- Attempting to delete a non-existent memory returns 404
- Consider using TTL (time-to-live) when creating memories for automatic expiration
- Deleted memory IDs cannot be reused
